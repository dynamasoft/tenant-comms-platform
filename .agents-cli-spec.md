# Agent Spec — Tenant Communication Platform

## Problem
Local MVP (Kaggle capstone) where a Florida landlord uploads a lease/addendum, then asks
questions about tenant situations (unpaid rent, notices, utilities, lease breaks,
abandonment, maintenance access, unauthorized occupants). The agent acts as an intelligent
landlord assistant combining lease retrieval, Florida landlord-tenant law lookup, risk
analysis, deadline calculation, and tenant-message drafting.

## Capabilities
An ADK 2.0 graph `Workflow` with nodes:
`intake → lease_retrieval → florida_law (MCP) → risk_analysis → timeline → message_draft → final_response`.

## External data / tools
- **MCP server** (local, stdio) for Florida landlord-tenant law lookup (Ch. 83 Part II).
- **One explicit ADK tool**: `timeline_calculator` (deterministic notice-deadline math).
- Everything else = graph reasoning + MCP context + LLM generation (no other tools).

## Model
Vertex AI (ADC), project `sanguine-method-458311-b0`, location `global`,
`gemini-flash-latest` (scaffold default; verified working).

## Safety constraints
- Always cite lease passages used.
- Separate lease findings from Florida law findings.
- Explicitly state uncertainty; avoid legal overconfidence; never claim to be a lawyer.
- Always append: "This is not legal advice. Verify with a Florida attorney or local court
  before taking legal action." (appended in code, not left to the LLM).
- Favor safe, conservative guidance.

## UI
Local Streamlit app (`app.py`): file upload (PDF/DOCX/TXT, in-memory only), scenario
textarea, tone dropdown (friendly/professional/firm/final warning), Analyze button, and 7
output sections.

## Testing & eval
- pytest: deterministic unit tests + 3 offline e2e scenarios (mocked model + MCP).
- Custom lightweight `evals/run_evals.py` (heuristic scoring) + native `agents-cli eval`.

## Deployment
Prototype only (`--prototype`, deployment_target=none). Add later via `scaffold enhance`.
